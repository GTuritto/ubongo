# v0.5 Phase 07 — The contract and identity

Phase 07 of [Plans/v0.5-trust-protocol.md](v0.5-trust-protocol.md), the **final** phase of the
trust protocol. Branch `v0.5/07-contract-identity` → fills the **v0.5.7** slot (CI derives the
version from the branch name; do not hand-edit VERSION). Deliberately the lightest phase: two
small, concrete capabilities plus a recorded set of deferrals. After this, the trust protocol is
complete.

## The shape

Two buildable pieces, both additive, plus an explicit "designed-but-deferred" record.

1. **Verbosity per domain — as governance config.** Today every response's length is whatever the
   persona produces. Phase 07 adds a legible, per-domain verbosity knob: a `verbosity:` block in
   `governance.yaml` mapping a domain (keyed by classifier `task_type` / `intent`) to a level
   (`terse | normal | deep`), with a default. The level is threaded into prompt assembly
   (`context.build_system_prompt`) as one extra directive line, so it shapes the answer without
   touching orchestration. **Manual first** — the config is hand-edited and surfaced read-only via
   `/verbosity`; a per-turn override (`/verbose`, `/brief`) mirrors `/skill`'s one-shot pattern.
   Learned legibility (a GP-evolvable `verbosity:<domain>` target) is named here and **deferred**:
   the GP loop is the natural machinery, and because promotion stays human-approved (ADR-0006), the
   dangerous half of "learning the boundaries" is structurally impossible — but it is not this
   phase.
2. **Backup, portability, identity.** Identity rides on the Phase-02 layering: an instance *is* its
   SQLite file + vault + config + skill library. `ubongo backup [path]` writes a portable archive
   of exactly those (never `.env` — secrets stay out); restore is replaying that archive into a
   fresh checkout. There is no install log to replay (D1 from the trust plan: capabilities are the
   human-approved allowlist, which is itself config). **Grants do not migrate**: a restored/moved
   instance re-asks per capability class on its new envelope, so the backup either excludes the
   `grants` rows or the restore re-arms them.
3. **Designed-but-deferred, recorded not built.** Forking, naming-as-a-speech-act, and
   inter-instance skill exchange wait until a second instance has a reason to exist. When it does,
   authored skills already carry quarantine-and-approve (ADR-0013) as their import path. ADR-0022
   records these as deliberate non-goals so they are decisions, not omissions.

```text
verbosity:  governance.yaml -> context.build_system_prompt (one directive line)
            /verbosity (read), /verbose|/brief (one-shot override)
backup:     ubongo backup [path] -> archive(data/ubongo.db*, vault/, config/)  [no .env, grants re-arm]
            restore = unpack into a fresh checkout; migration replays config, not an install log
deferred:   fork / naming / inter-instance exchange  (ADR-0022, not built)
```

\* the `grants` posture on backup is decision D2 below.

## What changes

1. A `verbosity:` block in `config/governance.yaml` (default + per-domain levels) and a
   `governance/verbosity.py` resolver (`level_for(classification) -> str`), threaded into
   `context.build_system_prompt` as a directive line. `/verbosity` (read the table) +
   `/verbose|/brief` (one-shot per-turn override, carried on the existing directives seam).
2. `ubongo backup [path]` (CLI via `oneshot`) — a timestamped `tar.gz` of the DB, vault, and
   config tree, excluding `.env`. A short `docs/` note on restore/migration (unpack into a fresh
   checkout; grants re-arm).
3. **ADR-0022** records the contract: verbosity is governance config (legible, manual-first,
   GP-evolvable later under the human floor); an instance is its data + config, backups carry no
   secrets and no portable grants; fork/naming/exchange are deferred.
4. CHANGELOG `## v0.5.7`, smoke section **X**, STATE/CONTEXT updates. The trust protocol is marked
   complete.

## Decisions to settle before code

- **D1 — Verbosity config home + key.** *Recommend* a `verbosity:` block in **governance.yaml**
  (verbosity is a legibility/governance concern, and governance.yaml is already the single home for
  decision config), keyed by classifier **`task_type`** with an `intent` fallback and a `default`.
  Levels `terse | normal | deep`. Surfaced read-only via `/verbosity`; the per-turn override is a
  one-shot directive, not a persisted setting.
- **D2 — Grants on backup.** *Recommend* the backup **includes** `data/ubongo.db` whole (it is the
  instance's memory) but the **restore re-arms grants** — the first connector turn on the restored
  instance asks again (matches "grants do not migrate"; the new envelope is a new trust boundary).
  Simplest honest rule: backup is a faithful copy; *migration to a new envelope* is the act that
  re-asks, enforced by clearing `grants` on restore (a `ubongo restore --fresh-grants`, default
  on). Alternative considered: strip grants at backup time — rejected because a same-machine
  disaster-recovery restore would then needlessly re-ask.
- **D3 — Learned verbosity now or deferred.** *Recommend deferred.* Ship manual config only; add a
  `verbosity:<domain>` evolvable target in a later phase. The plan is explicit that verbosity is
  manual-first; wiring the GP target now spends budget on the one phase meant to be light.

## QA test plan

### Acceptance criteria (exit = all checked)

- [ ] **AC-1 Verbosity threads into the prompt.** `build_system_prompt` for a turn whose
      `task_type` maps to `terse` includes the terse directive; `deep` includes the deep one; an
      unmapped domain falls to the default. No orchestration change — one directive line.
- [ ] **AC-2 Manual + legible.** `/verbosity` prints the live table from governance.yaml; editing
      the block + `/reload` changes the next turn's directive. Nothing learns or self-edits it.
- [ ] **AC-3 One-shot override.** `/brief` (or `/verbose`) sets the next turn's level and clears
      after, mirroring `/skill`; it overrides the domain default for that turn only.
- [ ] **AC-4 Backup is portable + secret-free.** `ubongo backup` writes an archive containing
      `data/ubongo.db`, `vault/`, and `config/` (settings, governance, jobs, personas, skills) and
      **not** `.env`; unpacking into a fresh checkout reproduces the instance (same memory, config,
      skills).
- [ ] **AC-5 Grants re-arm on a new envelope.** A restored instance (or `restore --fresh-grants`)
      has no active grants; the first connector turn asks again. Same-machine restore without the
      flag keeps them.
- [ ] **AC-6 Deferrals are recorded.** ADR-0022 names fork / naming / inter-instance exchange as
      deferred with the import path (quarantine-and-approve) for when they land.
- [ ] **AC-7 Additive.** Every channel + orchestration + the other daemons are byte-unchanged; the
      verbosity directive degrades to `normal` when the block is absent.
- [ ] **AC-8 Full suite green** with `tests/test_governance_verbosity.py` + `tests/test_backup.py`;
      ADR-0022 + CHANGELOG `## v0.5.7`; the trust protocol marked complete in STATE.md.

### Regression plan (layered)

1. **Unit.** `verbosity.level_for` (mapping, fallback, default, missing block → normal); the
   directive injection in `build_system_prompt`; the one-shot override on the directives seam; the
   backup archive contents (tmp dir, assert membership + `.env` absence) and the restore
   grant-rearm.
2. **Deterministic surface.** `/verbosity` table; `/brief`/`/verbose` in the help banner;
   `ubongo backup` to a temp path → a non-empty archive; a malformed `verbosity:` block → friendly
   fallback to normal.
3. **Live.** A `deep`-mapped technical question reads longer/structured; a `terse`-mapped one reads
   short; `/brief` shortens a normally-deep turn.
4. **Cumulative playbook** 0–21/A/P/M/C/L/E/S/T/U/V/W stay the contract; new section **X** is this
   phase's surface.

### Risks

| Risk | Impact | Mitigation |
| --- | --- | --- |
| Verbosity directive fights the persona voice | muddled tone | it is one additive line under the persona, not a rewrite; default `normal` is a no-op; AC-1/AC-7 |
| Backup leaks `.env` | secret exposure | the archive builder explicitly excludes `.env`; AC-4 asserts its absence |
| Restored grants silently re-authorize on a new envelope | over-broad consent after a move | grants re-arm on restore by default (D2); AC-5 |
| "Identity" scope-creeps into fork/naming now | the light phase balloons | those are recorded as deferred (ADR-0022), not built; AC-6 |
| Learned verbosity sneaks in | spends budget, risks the legibility story | deferred (D3); manual config only this phase |

## Why this fits the architecture

Verbosity is one more legible governance config consulted at prompt-assembly time — the same shape
as the live-swap reads, with no new seam and the human floor (ADR-0006) intact. Backup is a copy of
the data + config an instance already is; it invents no install log and carries no secrets or
portable consent. Nothing here adds a path around a governed seam; it is the trust protocol's quiet
final settling, not a new subsystem.
