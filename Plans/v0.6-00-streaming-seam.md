# v0.6 Phase 00 — The streaming seam

Phase 00 of [Plans/v0.6-live-console.md](v0.6-live-console.md), opening the v0.6 live-console line.
Branch `v0.6/00-streaming-seam` → **v0.6.0** (CI derives the version from the branch name). This
phase ships the one genuinely new primitive — **per-turn event streaming** — plus a bare console
that prints the live event log. The rich panels (roster, activity, approval, sources) are later
phases; this is the transport they ride on.

Recommendations accepted (from the umbrella plan): **D1 single-flight** (one active turn at a time,
zero correlation plumbing), **D2 token-streaming deferred** (this phase streams pipeline *events*,
not answer tokens), **D3 retire Streamlit later** (Phase 05, untouched here).

## The shape

A sixth channel, additive over `channel.run_turn` — same pattern as MCP/Telegram. The new part is
liveness: the turn runs in a background thread while the named bus events are forwarded to the
browser over Server-Sent Events (SSE). No orchestration change; this is a handler on the named
events (the CLAUDE.md rule for new behavior), not a pipeline edit.

- **`web/console/stream_bridge.py`** — the channel-free, HTTP-free, unit-testable core. `start_turn`
  registers handlers on the pipeline events (`after_classify`, `after_plan`, `agent_started/
  completed/failed`, `before_govern`, `after_compose`, `after_send`), runs `channel.run_turn` on a
  background thread, and forwards a JSON-safe summary of each event into the session's queue;
  `event_stream(stream_id)` drains the queue as SSE frames until a terminal `__end__`. **Single-
  flight**: a second `start_turn` while one is live is refused (the console serves one user at one
  keyboard). The console server starts no daemons, so only the turn's own events fire — no leakage.
- **`web/console/app.py`** — the only module importing FastAPI/uvicorn (the optional `[console]`
  extra, lazy). `POST /turn` → `start_turn` → `{stream_id}`; `GET /stream/{id}` → the SSE response;
  `GET /` → a bare `console.html` that POSTs a turn and appends each streamed event to a `<pre>`.
- **`ubongo console`** entrypoint (uvicorn, LAN no-auth posture like web/MCP); ctl + start script.
- **ADR-0023** records the per-turn event-streaming seam: background-task turn + SSE drain of the
  event bus, single-flight v1, no-auth LAN, no orchestration bypass.

## What changes

1. `web/console/` (`stream_bridge.py`, `app.py`, `console.html`) and the `ubongo console` entrypoint.
2. Optional extra `[console]` (fastapi + uvicorn), imported only under `web/console/`; the core +
   suite run without it (the streamlit/mcp/telegram precedent). ctl `start|stop|status console`,
   `start-ubongo-console.sh`.
3. ADR-0023; CHANGELOG `## v0.6.0`; smoke section **Y**.

## QA test plan

### Acceptance criteria (exit = all checked)

- [ ] **AC-1 No-bypass.** A console turn runs through `channel.run_turn` → `master.handle`;
      classified, governed, persisted like a typed turn. The stream is observation only.
- [ ] **AC-2 Events stream in order.** The forwarded sequence matches the pipeline event order for
      the turn; the final frame is a terminal `__end__` carrying `ok`.
- [ ] **AC-3 Single-flight.** A second `start_turn` while one is active is refused (returns None);
      after the active turn ends, a new one is accepted.
- [ ] **AC-4 Handlers are cleaned up.** After a turn (success or exception) every registered handler
      is unregistered — no leak into the next turn; an exception still emits a terminal frame.
- [ ] **AC-5 SDK isolation.** FastAPI/uvicorn import only under `web/console/`; the core + suite are
      green without the `[console]` extra (the bridge tests drive the forwarder directly, no HTTP).
- [ ] **AC-6 Additive.** REPL/one-shot/web/MCP/Telegram + orchestration byte-unchanged; no new core
      dependency.
- [ ] **AC-7 Ops.** `ubongo console` serves the LAN; missing extra → friendly hint, rc 1; ctl
      start/stop/status console works.
- [ ] **AC-8 Full suite green** with `tests/test_stream_bridge.py`; ADR-0023 + CHANGELOG `## v0.6.0`.

### Regression plan (layered)

1. **Unit (the bridge, no HTTP).** With `channel.run_turn` mocked to dispatch a scripted event
   sequence: `start_turn` forwards them as ordered frames + a terminal `__end__` (AC-2); a second
   concurrent `start_turn` is refused (AC-3); an exception in the turn emits a terminal error frame
   and unregisters handlers (AC-4); the registered handler set is empty after the turn.
2. **Deterministic.** `ubongo console` without the extra → hint + rc 1; help/usage.
3. **Live (manual).** Open the console, send a turn, watch the events stream; `curl -N /stream/<id>`.
4. **Cumulative playbook** 0–21/A/P/M/C/L/E/S/T/U/V/W/X stay the contract; new section **Y**.

### Smoke playbook section (lands as section Y)

Y.1 `ubongo console` without the extra → friendly hint, rc 1 · Y.2 unit: a scripted turn streams its
pipeline events in order + a terminal frame · Y.3 unit: single-flight refuses a second concurrent
turn · Y.4 unit: a turn exception emits a terminal error frame and cleans up handlers · Y.5 live:
the browser console streams a real turn (LAN) · Y.6 full pytest.

### Risks

| Risk | Impact | Mitigation |
| --- | --- | --- |
| A background-turn exception hangs the stream | console stuck "working" | the runner wraps the turn, always emits a terminal frame + sentinel, always unregisters (AC-4) |
| Concurrent turns mix events | wrong stream content | single-flight refuses a second turn (AC-3); the console server runs no daemons, so no other dispatcher fires |
| FastAPI import leaks into core | suite needs the extra | imported only under web/console/; AC-5 |
| No auth on the console | anyone on the LAN drives it | explicit non-goal (home-LAN, like web/MCP); ADR-0023 records it; the egress envelope is the boundary |
