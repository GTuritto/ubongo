## Architectural Outline (the new v0.1)

### The Core Loop

```text
CLI input
    |
    v
Master Agent.classify(message)
    |   -> intent, tone, risk, suggested workflow
    v
Master Agent.plan(intent, tone, risk)
    |   -> Workflow (set of agents + execution mode)
    v
Workflow Runner.execute(workflow)
    |   -> spawn agents, route messages between them, aggregate output
    v
Governance.gate(output, risk)
    |   -> approve / request confirmation / reject
    v
Response Composer.compose(output, persona)
    |   -> final user-facing text
    v
Queue + delivery + memory write + vault projection
```

### The Master Agent

- Owns task decomposition, agent creation/destruction, workflow coordination.
- Owns the **Decision Matrix**: `Decision = f(Intent, Risk, Confidence, Context, Preferences, Reversibility)`.
- Decides execution mode per workflow (sequential / parallel / competitive / collaborative / debate / speculative).
- Spawns Persona Agents for user-facing voice; spawns Worker Agents for capabilities.

### Worker Agents (full vision set)

Each is a class implementing a common `Agent` protocol: `run(input, context) -> Result`. All disposable; durable intelligence lives in memory, workflows, and policies.

- **Research Agent**: search-style retrieval + synthesis. v0.1 backend: LLM-only retrieval over conversation memory and vault. v0.2+ adds web/calendar/email skills.
- **Coding Agent**: code generation, refactoring, review. Backend: strong coding model.
- **Evaluator Agent**: validates other agents' output against criteria; produces a confidence score. Used by Master Agent for gating.
- **Repair Agent**: detects stuck/failed workflows, replaces agents, triggers retries with different models, performs rollbacks.
- **Memory Agent**: single writer to long-term memory; mediates fact extraction, summary persistence, vault writes. No other agent writes durable state.
- **Critic Agent**: contrarian / brutal analysis. Argues against the prevailing answer to surface gaps.
- **Execution Agent**: invokes shell commands, scripts, external APIs. Sandboxed (Phase 16). v0.1 ships a constrained-bash skill, not arbitrary execution.
- **Persona Agents**: Architect / Operator / Casual / etc. Inherit from a base, differ in voice. The user-facing surface; assemble final response from worker output.

### Execution Modes

All six implemented in v0.1:

1. **Sequential**: A â†’ B â†’ C.
2. **Parallel**: A | B | C, results aggregated by Evaluator.
3. **Competitive**: same input, multiple agents, best output picked by Evaluator.
4. **Collaborative**: each agent owns a subtask, results merged.
5. **Debate**: two agents argue opposing positions, third (Critic) synthesizes.
6. **Speculative**: cheap fast agent runs first; strong agent validates/replaces in background. User sees the cheap result; if validation contradicts, system corrects within the same session.

### Governance Layer

- **Risk evaluation**: per-workflow tag (`low` / `medium` / `high` / `destructive`).
- **Confidence**: from Evaluator Agent.
- **Reversibility**: declared per skill.
- **Decision rules**: `auto` / `ask_clarification` / `require_approval` / `escalate` / `reject`.
- **v0.1 approval primitive**: text confirmation in the CLI. Phase 14 ships this; later phases may extend.

### Self-Improvement (GP Loop)

- **Generation step**: Master Agent (or its evolution-controller submodule) emits N variants of a target (prompt, routing rule, tool chain).
- **Sandbox evaluation**: variants run against a held-out conversation sample. Sample is curated from past sessions, anonymized.
- **Fitness function**: configurable weighted sum of success rate, cost, latency, hallucination rate, user-correction rate.
- **Selection + lineage**: top variants survive; ancestry stored in `evolution_lineage` table.
- **Promotion**: best variant queued for human approval via `/improvements`. User approves or rejects per item.
- **Continuous loop**: runs as a background asyncio task at low priority; throttled to avoid hammering OpenRouter.
- **Targets**: prompts (persona prompts, classifier prompt, skill prompts), routing rules (the YAML), tool chains (which agents run for which intent), retry strategies.
