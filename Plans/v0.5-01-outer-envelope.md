# v0.5 Phase 01 — The outer envelope

Phase 01 of [Plans/v0.5-trust-protocol.md](v0.5-trust-protocol.md) (Decision 3 + Amendment 1).
Deployment infrastructure only: zero `src/` LOC. Branch `v0.5/01-outer-envelope`.

## What this phase builds

A network envelope around the Linux/Pi deployment so that what leaves the machine is
enumerable and enforced at the network layer, not at the model's discretion. SECURITY.md
already names the hole: the Connector's tool arguments leave the machine controlled only by
per-server `risk`/`enabled` flags, and the sandbox's network story for everything else is
"the allowlist has no network programs," not "the network is closed."

Three layers, all outside `src/`:

1. **A dedicated `ubongo` user.** The services run as this user; its UID is the key the
   firewall matches on. Lingering enabled so user units survive logout/reboot.
2. **Podman rootless quadlets** (`deploy/envelope/`): a `Containerfile` building the repo
   with uv, and `ubongo-web.container` / `ubongo-mcp.container` quadlets replacing the bare
   systemd units on enveloped hosts. `.env` is mounted **read-only**; `data/` and `vault/`
   are writable volumes; the container publishes only the service port.
3. **An nftables egress allowlist keyed on the UID** (`nftables-ubongo.conf` +
   `refresh-egress.sh` + a systemd timer): for packets with `meta skuid ubongo`, allow
   loopback, DNS to the local resolver, and TCP/443 to the resolved addresses of the
   allowlisted hosts — today exactly `openrouter.ai` (every model and embedding call goes
   through LiteLLM → OpenRouter) plus any host declared in `settings.yaml::mcp.servers`.
   Everything else for that UID is rejected. nftables cannot match FQDNs, so the refresh
   script resolves the hostnames into named sets on a timer; a resolution failure leaves the
   previous set in place (fail-closed against *new* hosts, never fail-open).

## Derivation of the allowlist

- `openrouter.ai:443` — `llm.complete` (LiteLLM) and `memory/embeddings.py` both route
  every call through OpenRouter (verified against `config/settings.yaml::models.*` and
  `memory.embeddings.model`).
- Hosts parsed from `settings.yaml::mcp.servers.*.url` — currently `{}`; the documented
  loop-back example is `127.0.0.1` (covered by the loopback rule).
- Nothing else. The shell sandbox's program allowlist contains no network clients, so no
  other component originates egress.

## What this phase does NOT do (Amendment 1)

- **macOS is not enveloped.** Podman quadlets and nftables are Linux-only; the macOS dev
  machine keeps the LAN-trust posture. The ADR names this asymmetry instead of implying a
  universal guarantee.
- No change to `sandbox.py`, governance, or the Connector. The envelope is defense in depth
  *underneath* ADR-0016, not a replacement for the grant registry (Phase 05).

## The terrarium (falls out of this phase)

The same quadlet with `INSTALL.md`'s terrarium overrides — a looser allowlist set, inert
API keys, `UBONGO_DISABLE_*` switches off, daemons resumed — is the growth experiment
described in the v0.5 plan. Documented as a variant, not a separate artifact.

## Testing plan

This phase ships no Python, so the pytest gate is regression-only (suite stays 960 green).
Local validation is honest about its limits on macOS: shell syntax (`bash -n`,
`zsh -n`), nftables/quadlet files are reviewed structurally but can only be *executed* on
the Linux host. The playbook section carries the on-device verification:

- E.1: from inside the container, `https://openrouter.ai` is reachable (a real turn works).
- E.2: from inside the container, a non-allowlisted host is unreachable — fails at the
  network layer (connection refused/timeout from nft reject), not by program refusal.
- E.3: a Connector call to an unconfigured destination fails at the network layer.
- E.4: `.env` is read-only inside the container (`touch /app/.env` fails).
- E.5: the egress sets refresh on the timer; a resolution failure logs and keeps old sets.

Exit criteria (from the v0.5 plan): on the Linux deployment, Ubongo cannot reach a
non-allowlisted host, verified by attempt; ADR recorded naming the macOS gap.
