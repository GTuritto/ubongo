# v0.5 Phase 02 — Split the store (+ shared judgment parsing)

Phase 02 of [Plans/v0.5-trust-protocol.md](v0.5-trust-protocol.md); the architecture review's
candidate 1 (trigger fired) with candidate 3 riding along (Amendment 2). Branch
`v0.5/02-store-split`. **Behavior-free refactor**: no schema change, no SQL change, no new
behavior — functions move, callers re-import, tests split along the same seams.

## Baseline (measured 2026-06-12 on `main`)

- `memory/store.py`: **1,990 lines, 92 functions** — the largest module in the tree.
- `memory/trace.py`: 112 lines (read-side views only; the 167-line builder lives in store).
- `tests/test_memory_store.py`: 391 lines, one module for six concerns.
- `src/` total: 15,401 lines. Suite: 960 green.
- Cross-package private imports: `authoring/{loop,sandbox}.py → evolution.sandbox.CallBudget`,
  `authoring/commands.py → evolution.commands._diff_preview`.

## The seams (from the review, confirmed against callers)

| New home | Cluster moved from store.py | Owning callers |
| --- | --- | --- |
| `memory/store.py` (stays) | bootstrap/connection, conversations, messages, summaries, sessions, `recall()` | master, runner, repl, memory agent — the per-turn core |
| `memory/trace.py` (absorbs) | workflow_runs, agent_runs, governance_decisions, repair_runs CRUD + `last_n_workflow_runs()` (the builder joins the views already defined there) | master, runner, repl `/trace`, profiling |
| `memory/evolution_state.py` (new) | evolution_lineage, evolution_evaluations, evolution_runs, evolution_state, pending_promotions, active_evolutions | evolution/* , context live-swap read, router live-swap read |
| `memory/authoring_state.py` (new) | authored_skills, authoring_runs, authoring_state | authoring/* |
| `memory/index_state.py` (new) | embedding_meta, vault_links, vault_state, vault_conflicts (+ `messages_by_ids`) | embeddings, graph, vault, vault_watch, memory commands |
| `src/ubongo/evaluation.py` (new, candidate 3) | (from the sandboxes, not the store) `CallBudget`, judgment JSON parsing/fence stripping, `diff_preview` | evolution/sandbox+commands, authoring/sandbox+commands+loop |

Rules of the move: each function keeps its body byte-for-byte wherever possible; the seam
modules import `connection()` from `store` (one bootstrap, one connection — ADR-0002's
single-writer rule is about *writers*, not files, and is untouched); no re-export shims —
callers update their imports so the dependency graph tells the truth.

## QA test plan (prepared with /qa-test-planner)

- [ ] **AC-1 Behavior parity:** full suite green; **no assertion changed or deleted** — only
      import lines and test-module homes move.
- [ ] **AC-2 Seam inventory:** the five store modules exist with their clusters;
      `store.py` ≤ ~800 lines; no function exists in two places; no re-export shims.
- [ ] **AC-3 No SQL drift:** `schema.sql` untouched; the moved functions' SQL strings are
      byte-identical (reviewable in the diff as pure moves).

      (`test_agents_memory.py::test_strict_mode_blocks_non_memory_writer`) passes unmodified;
      no new writer of messages/vault/embeddings appears.
- [ ] **AC-5 LOC honesty:** `src/` total ≤ 15,401 (net ≤ 0; target negative via candidate 3).
- [ ] **AC-6 One evaluation-common module:** `ubongo.evaluation` owns CallBudget + judgment
      parsing + diff preview; `grep "from ubongo.evolution" src/ubongo/authoring/` → no hits.
- [ ] **AC-7 Test seams:** `test_memory_store.py` splits into per-seam test modules with the

- [ ] **AC-8 Docs current:** STATE.md structural-drift table gains the split row; playbook
      gains section S; CONTEXT.md untouched unless a new term is coined.

### Regression plan (layered, checkpointed)

1. **Per-seam checkpoint:** move one seam at a time, run `uv run pytest -q` after each move;
   commit per seam (trace → evolution → authoring → index → evaluation-common → test split).
   A red suite at any checkpoint stops the line — fix or revert that seam before proceeding.
2. **Deterministic surface pass (P1):** the commands that read through moved clusters:
   `/trace`, `/decisions`, `/queue`, `/profile`, `/recall`, `/conflicts`, `/audit`,
   `/skill-candidates`, `/evolution status`, `/optimize` (list) — all on a fresh DB,
   asserted by grep exactly as the cumulative smoke does.
3. **Live mini-sweep (P0):** one governed turn (rows in all four trace tables + `/trace 1`
   renders), `/optimize persona:casual` (lineage via the new module), approval-gate y path
   (governance update via the new home), `/recall` (embedding meta via index_state).
4. **Cumulative playbook:** the existing sections 0–21/A/P/M/C/L stay the regression
   contract; new section S below is the phase's own acceptance surface.

### Risks

| Risk | Impact | Mitigation |
| --- | --- | --- |
| Hidden import cycle (seam module ↔ store) | suite-wide failure | seam modules import only `store.connection`; store never imports a seam module |
| A caller missed in the re-import sweep | NameError at runtime in a rarely-exercised path | grep-driven caller inventory per cluster before each move; the deterministic surface pass covers every read command |
| Test split silently drops cases | false green | AC-7 counts tests before/after; the gate is count-equal-or-higher |
| live-swap read paths (context/router) regress | evolution behavior changes | their tests are in the suite; the live mini-sweep approves a promotion read path via `/optimize` + status |
